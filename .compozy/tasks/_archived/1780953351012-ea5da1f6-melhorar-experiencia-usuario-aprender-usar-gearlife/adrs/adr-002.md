# ADR 002: Use a shared content component for the info modal and How It Works page

Date: 2026-05-30
Status: Accepted

## Context

The current app uses a dedicated `InitialInfo` component for modal help and a separate standalone `/como-funciona` page. The business requirement is to keep a quick-start modal while synchronizing the learning content with the full reference page.

## Decision

Extract the shared learning content into a single React component that both the `InitialInfoModal` and `ComoFuncionaPage` consume. Keep the modal wrapper and the page wrapper separate, but reuse the same content structure and media assets.

## Alternatives Considered

- **Keep separate modal and page content files**: simpler to implement initially, but increases the risk of stale or diverging text and images.
- **Use a shared data model with separate presentation components**: makes content reusable but adds unnecessary indirection for this small feature.

## Consequences

- Updates to the learning content only need to happen in one component, reducing drift between modal and page.
- The `/como-funciona` page and help modal remain visually and textually consistent.
- The codebase gains a clear shared component boundary for learning content.
