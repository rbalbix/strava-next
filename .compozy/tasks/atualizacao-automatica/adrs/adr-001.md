# ADR 001: Implementação de Atualização Automática da UI via Polling Inteligente

## Status
Accepted

## Date
2026-06-08

## Context
Atualmente, quando o GearLife processa uma nova atividade via webhook do Strava, o usuário precisa atualizar a página manualmente ou fazer logout/login para visualizar os dados atualizados. O objetivo é que a interface reflita essas mudanças de forma "silenciosa" e automática.

Foi considerada a utilização de WebSockets ou Server-Sent Events (SSE) para tempo real instantâneo, porém essas abordagens trazem custos de execução elevados em ambientes serverless (Vercel) ou exigem serviços externos pagos (Pusher/Ably).

## Decision
Utilizaremos **Polling Inteligente** com intervalo de **5 segundos** enquanto a aba do navegador estiver ativa e visível.

A implementação deverá:
1. Consultar o endpoint de Dashboard (ou um endpoint de verificação leve) a cada 5 segundos.
2. Atualizar as estatísticas nos cards e verificar se novos alertas de manutenção devem ser disparados.
3. Desativar completamente as requisições quando a aba estiver em segundo plano (Page Visibility API) para economizar recursos e bateria.
4. Utilizar revalidação ao focar (Focus Revalidation) para garantir dados frescos assim que o usuário retornar ao app.

## Alternatives Considered
- **Conexão Contínua (SSE):** Descartada devido ao alto custo de faturamento por tempo de execução em serverless.
- **Serviços Externos (Pusher/Ably):** Descartados para evitar dependências de terceiros e custos adicionais de assinatura.
- **Revalidação apenas por Foco:** Descartada pois o usuário deseja ver a atualização acontecendo enquanto observa a tela ("instantâneo").

## Consequences
- **Prós:** Simplicidade de implementação, custo zero de infraestrutura adicional, experiência próxima ao tempo real.
- **Contras:** Leve aumento no tráfego de rede e acessos ao Redis (mitigado pela desativação em background).
