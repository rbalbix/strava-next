# 0002 — Armazenar limites de equipamento no Redis e usar modal independente de alerta

## Status

Aceito — define a persistência e a modal de alerta para limites de distância por equipamento.

## Contexto

GearLife já mantém dados de atleta, atividades e estatísticas no Redis. A nova funcionalidade requer que limites de distância por equipamento persistam entre sessões e dispositivos, e que alertas sejam exibidos de forma independente quando algum equipamento atingir o limite.

## Decisão

1. Armazenar os limites de distância no Redis com uma chave por atleta: `strava:equipment-thresholds:<athleteId>`.
2. Usar um modelo de dados aninhado por `gearId` e `equipmentId` para suportar limites por equipamento em cada bike/gear específica.
3. Expor um novo endpoint interno de API em `/api/app/equipment-thresholds` para ler e gravar os limites do usuário.
4. Reutilizar o dashboard atual para carregar `gearStats` e os limites quando a página for iniciada.
5. Implementar um modal independente de alerta no próximo login se houver algum equipamento com limite ultrapassado.

## Alternativas Consideradas

- Alternativa A — Armazenar os limites apenas no navegador (localStorage/sessionStorage).
  - Rejeitada: não persiste entre dispositivos nem entre navegadores do mesmo usuário.
- Alternativa B — Usar o endpoint de `remoteStorage` interno com chave única genérica.
  - Rejeitada: o endpoint atual exige `X-Internal-Api-Key` e não é adequado para dados do usuário autenticado.
- Alternativa C — Adicionar limites ao modelo `statistics` em vez de criar chave Redis separada.
  - Rejeitada: mistura configuração do usuário com dados de estatísticas calculadas e dificulta a gravação independente.

## Consequências

- A persistência fica disponível entre sessões e dispositivos para o mesmo atleta.
- O novo endpoint é pequeno e isolado, evitando mudanças arriscadas no processamento de estatísticas existentes.
- A modal independente mantém o alerta de limite separado do fluxo de detalhe do equipamento.
- O esquema de chaves Redis permite futura expansão para outras preferências de equipamento.
