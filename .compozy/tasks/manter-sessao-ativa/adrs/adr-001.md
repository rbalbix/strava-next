# ADR-001: Sessão Longa Ininterrupta com Renovação Transparente

## Status
Accepted

## Data
2026-06-18

## Contexto
O aplicativo atualmente possui uma duração de sessão extremamente curta para os cookies `strava_athleteId` e `strava_code` (300 segundos / 5 minutos). Isso causa uma experiência ruim para o usuário, que precisa refazer a autenticação via Strava OAuth frequentemente, mesmo que use o aplicativo de forma contínua. Precisamos aumentar o tempo de sessão mantendo a segurança e garantindo fluidez.

## Decisão
Adotaremos a **Abordagem 1: Sessão Longa Ininterrupta com Renovação Transparente**. 
As especificações incluem:
1. Aumentar a expiração padrão dos cookies `strava_athleteId` e `strava_code` para um período longo fixo de **30 dias** (2592000 segundos).
2. Se o cookie expirar ou se tornar inválido (por exemplo, se o usuário revogar o acesso ao app no painel do Strava), o aplicativo interceptará o erro nas rotas protegidas ou chamadas de API de forma silenciosa e redirecionará o usuário automaticamente para o fluxo de autorização do Strava.
3. Se a renovação via Strava for bem-sucedida, o usuário retorna ao Dashboard imediatamente sem precisar clicar em nada ou ver mensagens de erro.
4. O logout manual limpará os cookies imediatamente e exigirá nova reautorização no próximo acesso.

## Alternativas Consideradas
* **Alternativa A — Opção de "Lembrar-me":** Adicionar um checkbox na tela inicial. Rejeitado para manter o MVP simples e livre de elementos visuais desnecessários de formulários, estendendo a sessão de forma longa para todos por padrão.
* **Alternativa B — Sessão Deslizante (Sliding Session):** Atualizar o cookie a cada requisição ativa. Rejeitado por adicionar complexidade de escrita de cabeçalhos em todas as respostas de API sem um benefício expressivo comparado à duração fixa de 30 dias com fluxo de re-auth transparente.
* **Alternativa C — Sessão Longa Estrita com Redirecionamento para Início:** Enviar o usuário para a página estática de login ao expirar, exigindo o clique manual no botão de login. Rejeitado por adicionar atrito desnecessário à experiência do usuário.

## Consequências
* **Positivas:** Experiência de uso totalmente fluida, sem interrupções visuais ou alertas de erro para sessões expiradas. Aumento significativo na retenção e engajamento com o painel de métricas.
* **Negativas:** Mudança rápida e automatizada na tela de navegação durante o redirecionamento invisível de expiração a cada 30 dias ou após revogações de token.
