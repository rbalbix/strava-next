# ADR-002: Arquitetura de Tokens Semânticos

*   **Status:** Accepted
*   **Data:** 2026-06-12

## Contexto
O sistema de estilos atual da GearLife utiliza uma mistura de nomes descritivos (ex: `--light-blue`) e funcionais. Isso dificulta a manutenção de temas e a consistência visual ao adicionar novos componentes ou ajustar a estética global.

## Decisão
Substituiremos totalmente as variáveis de cores descritivas por **Tokens Semânticos**.
1.  As variáveis serão nomeadas de acordo com sua função na interface (ex: `--surface-primary`, `--text-on-surface`, `--brand-active`).
2.  Toda a referência a cores nos arquivos `.module.css` deverá utilizar exclusivamente esses tokens.
3.  A refatoração será completa, removendo os nomes antigos para evitar ambiguidade.

## Alternativas Consideradas
*   **Camada de Abstração:** Manter cores base e apontar tokens para elas. Rejeitado por aumentar a verbosidade do `globals.css` sem um ganho claro para o tamanho atual do projeto.
*   **Prefixação:** Rejeitado por manter o "legado" vivo e gerar confusão sobre qual variável usar.

## Consequências
*   **Positivas:** Código mais limpo e legível; facilidade extrema para trocar de tema ou ajustar tons globais; garantia de consistência.
*   **Negativas:** Esforço inicial de refatoração em todos os arquivos CSS do projeto.
