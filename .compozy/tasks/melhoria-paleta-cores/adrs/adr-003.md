# ADR-003: Estratégia de Implementação de Glassmorphism e Fallbacks

*   **Status:** Accepted
*   **Data:** 2026-06-12

## Contexto
O PRD exige uma estética de "vidro" (glassmorphism) para modernizar a interface. No entanto, o efeito de `backdrop-filter: blur` não é suportado por todos os navegadores ou pode ser desativado por questões de performance.

## Decisão
1.  **Classe Global Utilitária:** Criaremos a classe `.glass-effect` no `globals.css`. Isso garante que o efeito seja aplicado de forma idêntica em qualquer componente.
2.  **Fallback por Gradiente:** Para dispositivos que não suportam o efeito, utilizaremos um gradiente linear sutil (`linear-gradient`) para simular a profundidade e a translucidez, em vez de uma cor sólida simples.
3.  **Detecção de Suporte:** Utilizaremos `@supports (backdrop-filter: blur(1px))` no CSS para alternar entre o efeito real e o fallback.

## Alternativas Consideradas
*   **Opacidade Fixa:** Rejeitado por ser visualmente "pobre" e não transmitir a sensação de modernidade desejada.
*   **CSS Composition (Modules):** Rejeitado em favor da simplicidade da classe global para uma utilidade tão onipresente na nova UI.

## Consequências
*   **Positivas:** Interface moderna e "viva"; degradação graciosa em hardware antigo; fácil aplicação em novos componentes.
*   **Negativas:** Requer testes manuais e de regressão visual para garantir que o gradiente de fallback seja visualmente aceitável.
