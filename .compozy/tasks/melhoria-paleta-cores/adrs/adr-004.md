# ADR-004: Automação da Validação de Acessibilidade Cromática

*   **Status:** Accepted
*   **Data:** 2026-06-12

## Contexto
Garantir o contraste adequado em dois temas (claro e escuro) e com transparências (glassmorphism) é um desafio técnico. O PRD estabelece a acessibilidade como um objetivo central.

## Decisão
Implementaremos **scripts de validação automática** no processo de build/CI.
1.  O script analisará o arquivo `globals.css` e extrairá os pares de cores de tokens (ex: `--text-primary` sobre `--surface-primary`).
2.  Utilizará bibliotecas de cálculo de contraste (mirando no algoritmo APCA conforme sugerido no PRD) para validar se os tokens atendem aos níveis mínimos.
3.  O build falhará caso uma combinação de cores definida no tema não atenda aos requisitos de acessibilidade.

## Alternativas Consideradas
*   **Validação Manual:** Rejeitada por ser propensa a erros e difícil de manter à medida que novas cores são adicionadas.
*   **Style Guide Estático:** Rejeitado por não ser impeditivo; desenvolvedores podem ignorar o guia acidentalmente.

## Consequências
*   **Positivas:** Acessibilidade garantida por contrato de código; feedback imediato para os desenvolvedores.
*   **Negativas:** Overhead inicial para configurar o script de análise de CSS e os cálculos de contraste.
